# 四层定义卡

---

## E 经验层

**一句话**：带结构标签的材料库。

**节点结构**：

```
e_i = (content, source, mediation, process, validity, function)
```

| 维度 | 问的问题 | 可选值 |
|------|----------|--------|
| source | 这条经验从哪来？ | personal / observed / reported / derived |
| mediation | 是直接获得还是经过中介？ | direct / mediated |
| process | 加工到什么程度了？ | raw / organized / reflective |
| validity | 现在可靠吗？ | unverified / supported / contested / stable |
| function | 在结构里干什么？ | evidence / counterexample / anomaly / boundary |

**子集切片方式**：按任意维度过滤，如 `E_direct = {e ∈ E | mediation = direct}`

**关键特性**：
- 不是树状分类，是多维标签；同一节点可同时属于多个子集
- 整体偏高熵，但内部有显著层位差异
- 是知识化的候选池、思维激活的输入池、回写的更新起点

---

## K 知识层

**一句话**：带结构标签的规则库。

**节点结构**：

```
k_j = (content, scope, abstraction, justification, function, stability)
```

| 维度 | 问的问题 | 可选值 |
|------|----------|--------|
| scope | 覆盖多大范围？ | personal / domain / general |
| abstraction | 抽象到什么程度？ | case / rule / principle |
| justification | 靠什么成立？ | evidential / inferential / systematic |
| function | 结构功能是什么？ | descriptive / normative / procedural |
| stability | 现在稳不稳？ | tentative / supported / stable / contested |

**子集切片方式**：同经验层，按维度过滤

**关键特性**：
- V1 的"个体/专业/普遍"三分法降级为 scope 维度的一部分
- 是经验压缩后的第一次结构稳定化
- 向经验层和哲学层双向张开

---

## T 思维层

**一句话**：五级递进的运算链。

**与经验层、知识层的本质区别**：不以"内容属性"为主结构，以"递进梯度"为主结构。

**五级梯度**：

```
T_repr → T_det → T_disc → T_dial → T_arch
                                        ↓
                           T_arch → T_disc → T_det → E/K（回落闭合）
```

| 级 | 名称 | 干什么 | 局限 |
|----|------|--------|------|
| T_repr | 表象 | 抓印象、建联接 | 漂浮、偶然 |
| T_det | 定型 | 划边界、分类、对象化 | 僵化、早封闭 |
| T_disc | 推演 | 规则链、因果链、论证 | 框架内封闭 |
| T_dial | 辩证 | 找反例、暴露矛盾、修正 | 只拆不建 |
| T_arch | 统摄 | 形成主线、重排核心 | 若不回落则空泛 |

**节点最小写法**（可选）：

```
t_s = (level, task, limit, upward_role, downward_role)
```

**关键特性**：
- "判断"不是独立第六级，而是各级的闭合动作
- 升熵/稳熵/降熵/回写是运行效应，不是层级本身

---

## P 哲学层

**一句话**：认知架构对自身的宪法定义。

**与前三层的本质区别**：不处理具体输入对象，处理架构自身。

**四段结构**：

```
P0 → PΔ → PI → PΣ
```

| 段 | 名称 | 干什么 | 不是什么 |
|----|------|--------|----------|
| P0 | 入口清障 | 去假象/偏见/旧体系投射 | 不是主梯度，是门 |
| PΔ | 差异层 | 哪些差异合法、必须保留 | 不是让差异失控 |
| PI | 同一层 | 保证稳定性、非矛盾、可追溯 | 不是强行统一 |
| PΣ | 综合统一 | 差异与同一的更高组织 | 不是消灭差异 |

**节点最小写法**（可选）：

```
p_h = (level, principle, role, constraint, integration)
```

**关键特性**：
- 对整套 E/K/T 施加宪法性约束，不只是"最高抽象层"
- 三种宪法作用：差异合法化(PΔ)、稳定性定义(PI)、综合统一(PΣ)

---

## 四层关系速览

**总序**（典型趋势，不是绝对）：

```
H_E > H_K > H_T > H_P  （经验层最高熵，哲学层最低熵）
```

**跨层接口**：

| 方向 | 接口 | 类型 |
|------|------|------|
| E → K | `Build_K`, `K_potential` | 生成（知识化） |
| E → T | `Act_ET` | 激活 |
| K → T | `Act_KT` | 激活 |
| P → E | `GateFit_EP` | 门控 |
| P → K | `GateFit_KP` | 门控 |
| P → (E,K,T) | `Constitution` | 宪法 |
| T_arch → T_det → E/K | — | 回落闭合 |
