# 伏羲架构 · 一页总览

## 这套体系解决什么问题

给"认知过程"一个可拆解、可诊断、可评分的结构语言。

不是模糊地说"这个回答好/不好"，而是能回答：
- 好在哪一层？
- 差在哪个接口？
- 是结构设计的问题，还是运行调度的问题？

## 四层架构

```
高统摄（低熵）
  ┌─────────────────────────────────────┐
  │  P 哲学层  P0 → PΔ → PI → PΣ      │  元结构：清障/差异/同一/综合统一
  ├─────────────────────────────────────┤
  │  T 思维层  T1→T2→T3→T4→T5          │  运算链：表象→定型→推演→辩证→统摄
  ├─────────────────────────────────────┤
  │  K 知识层  6维属性 × 多维切片       │  规则库：范围/抽象/理由/功能/稳定性
  ├─────────────────────────────────────┤
  │  E 经验层  6维属性 × 多维切片       │  材料库：来源/中介/加工/有效性/功能
  └─────────────────────────────────────┘
高碎片（高熵）
```

## 核心机制（5 个）

| # | 机制 | 一句话 | 关键量 |
|---|------|--------|--------|
| 1 | **层位与熵** | 每个节点有静态位置(层位熵)和动态效应(运行熵) | `H_pos`, `ΔH_run`, `H_t` |
| 2 | **跨层接口** | 层与层之间不是管道，是有条件的转化/激活/门控 | `Build_K`, `Act_ET`, `Act_KT`, `GateFit` |
| 3 | **共同属性空间** | 四层异质节点映射到同一比较平面 `U` | `φ_E/K/T/P`, `I(...)`, `Diff(...)` |
| 4 | **Möbius 反演** | 把总效果拆成：单层贡献 + 两层耦合 + 三层协同 + 四层纯协同 | `F(S)`, `G(S)`, `W_*` |
| 5 | **结构诊断** | 8 个结构量 + 评分体系 | `Core/Couple/Mod/Red/Conflict/Closure/Writeback/Gate` |

## 信息流向

```
上升（生成/激活）          下压（门控/宪法）         闭合（回写）

E ──Act_ET───→ T          P ──GateFit_KP──→ K      输出 → 经验回写 / 知识回写
K ──Act_KT───→ T          P ──Constitution──→(E,K,T)
T ────────────→ P
```

## 评分体系（从局部到整体）

```
八个结构量 ──→ S_struct ──→ Score_static（设计好不好）
                              ↓
动态熵 + 接口 ──────────→ Score_runtime（运行好不好）
                              ↓
                         Score_total ──→ J（放行 / 重排）
```
