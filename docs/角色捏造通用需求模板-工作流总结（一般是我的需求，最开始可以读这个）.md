# 角色捏造通用需求模板｜工作流总结

> 说明：本文件用于总结本次“托马斯·内格尔角色”生产流程中的全部关键需求，供后续快速复用到其他角色。

---

## 一、总目标

目标不是做普通“人设提示词”，而是做：

**基于四层双向贯通架构的、可部署、可评估、可迭代的哲学家/思想家角色系统。**

角色必须同时满足：

1. 像这个人
2. 能运行
3. 能评估
4. 能修订
5. 能继续扩展到其他角色

---

## 二、总流程

## 第一步：先理解母体架构

必须以 `系统提示词-伏羲.md` 作为母体系统宪法。

核心要求：

1. 四层双向贯通：经验 → 知识 → 思维 → 哲学
2. 每轮有：归纳 + 演绎 + 检验 + 印证
3. 每个关键结论要有：证据 / 边界 / 反例
4. 信息不足或冲突时，允许降级到 `under_review`
5. 固定原原则不可删除、不可替换、不可降级

---

## 第二步：先看案例模板，再做新角色

优先参照已经打磨成熟的**最终交付版角色模板**，而不是简稿。

可复用规则：

1. 看其结构
2. 看其经验层密度
3. 看其知识层补厚说明
4. 看其输出协议、回写协议、违例规则

---

## 第三步：角色提炼必须先走“目录法”

要求顺序：

1. 先看本地资料目录
2. 通过目录理解这位哲学家的重要思想结构
3. 再 web search 补充外部资料
4. 最后回填角色画像

目的：

- 不能一上来凭印象写人
- 必须从该人物的著作结构、问题结构出发

---

## 第四步：必须做 web search 补充画像

web search 的补充目标包括：

1. 生平
2. 师承
3. 学术共同体
4. 外部评价
5. 代表性问题意识
6. 学界认知中的关键位置

作用：

- 让角色画像不仅有概念，还有历史位置、学术位置与问题谱系

---

## 三、角色画像质量标准

## 1. 经验层不能只是列标题

经验层尤其是“间接经验”必须写成：

1. 经验来源是什么
2. 这类经验是什么意思
3. 它如何塑造角色的问题意识
4. 它在什么场景下会被调用

禁止：

- 只写“长期研习某某”
- 只写“与某某互动”
- 只写经验清单，不解释作用

---

## 2. 知识层不能只是术语清单

每个重要概念至少要写清：

1. 定义
2. 体系作用
3. 常见误读
4. 适用边界

目标不是“让人看懂角色是什么”，而是：

**让模型知道何时调用什么、如何调用、哪里不能乱用。**

---

## 3. 思维层必须是“程序”，不是标签

不能只写：

- 会分析
- 会总结
- 会批判

必须写成：

1. 默认程序
2. 高质量判据
3. failure pattern
4. 高压追问下如何升级回答

---

## 4. 哲学层必须既有原则，也有门控

哲学层必须同时具备：

1. 扩展原则
2. 与母体固定原原则的绑定
3. 运行门控
4. 出错时的违宪判定

---

## 四、角色输出形态要求

角色生产必须分阶段推进：

1. **画像稿**
   - 先打厚四层画像

2. **准系统提示词**
   - 在画像稿基础上补：
     - 宪法继承
     - 四层结构
     - 连接协议
     - 组合调度机制
     - 中文表达协议
     - 输出协议
     - 回写协议
     - 高严重度违例

3. **最终交付版**
   - 静态与动态评估通过后再整理成最终版

---

## 五、评估要求

## 1. 静态评估

在没有真实对话日志时，先对文本结构本身做评估。

核心检查项：

1. 四层顺序是否清楚
2. 层间连接是否明确
3. 层内主次是否明确
4. 是否有组合机制
5. 是否有闭环回写
6. 是否有规则与审核
7. 固定哲学预设是否完整生效

---

## 2. 动态评估

动态评估必须基于：

- web search 找到的专业问题
- 真实学术来源中的现成问法

禁止：

- 纯凭印象自己编测试题

---

## 六、动态评估题来源要求

优先来源：

1. 该人物相关课程大纲
2. SEP / IEP 词条里的问题表述
3. 权威书评中的核心争点
4. 出版社学术摘要中的核心问题
5. 学界介绍里的经典问法

目的：

- 让动态评估更接近专业场域，而不是泛泛常识问答

---

## 七、文件工作流要求

角色生产必须文档化、版本化、归档化。

建议文件阶段：

1. 画像稿 V1
2. 画像稿 V1 静态评估 V1
3. 准系统提示词 V2
4. 准系统提示词 V2 静态评估 V2
5. 动态评估题池（Web检索）
6. 准系统提示词 V2 动态评估 V1
7. 准系统提示词 V2 动态评估 V2
8. 最终交付版

要求：

1. 统一命名
2. 单独归档
3. 版本清晰

---

## 八、最终角色使用场景要求

最终角色不是只“像这个哲学家”，还必须能：

1. 面向博士与博士后阶段的大学教师
2. 进行思想对谈
3. 形成交锋与碰撞
4. 激发社科研究思路和灵感
5. 不只是复述知识
6. 不只是机械模仿语气

因此，一句话工作准则必须体现：

1. 高水平研究对话场景
2. 思想交锋
3. 追问推进
4. 研究启发性

---

## 九、通用角色生产模板

### A. 输入材料阶段

1. 指定母体架构
2. 指定参考案例（最好是最终交付版）
3. 提供该角色原著/资料目录

### B. 角色提炼阶段

1. 先看目录
2. 提炼思想主轴
3. web search 补生平/学术位置
4. 形成四层详细画像稿

### C. 系统化阶段

1. 从画像稿升级成准系统提示词
2. 补：
   - 宪法继承
   - 四层结构
   - 连接协议
   - 组合调度机制
   - 中文表达协议
   - 输出协议
   - 回写协议
   - 高严重度违例

### D. 评估阶段

1. 静态评估 V1
2. 修订
3. 静态评估 V2
4. web 检索动态题池
5. 动态评估 V1
6. 修订
7. 动态评估 V2

### E. 交付阶段

1. 生成最终交付版
2. 统一命名
3. 归档整理

---

## 十、最短复用原则清单

以后捏其他角色时，只要记住这 10 条：

1. 先看目录，再看外网，再写角色。
2. 角色必须挂在伏羲母体下面。
3. 参照的是高完成度最终交付版模板，不是简稿。
4. 经验层必须补厚，尤其是间接经验。
5. 知识层必须有定义、作用、误读、边界。
6. 思维层必须写成可执行程序。
7. 角色不是画像，要逐步做成可部署系统提示词。
8. 必须做静态评估 + 动态评估。
9. 动态评估题必须来自 web 搜索的专业问题，不自己编。
10. 最终角色要能服务高水平学术对谈，激发社科研究灵感。

---

## 十一、可直接复制使用的标准任务说明

以后若要快速捏其他角色，可直接使用这段说明：

> 请按我们之前的托马斯·内格尔流程来做：  
> 1）先看本地资料目录，用目录法提炼思想主轴；  
> 2）再 web search 补充该人物的生平、师承、学术位置与核心问题；  
> 3）先输出四层详细画像稿，不要急着出最终系统提示词；  
> 4）经验层尤其是间接经验必须补厚，不能只列标题；  
> 5）知识层要有定义、作用、误读、边界；  
> 6）思维层要写成可执行程序；  
> 7）再升级成准系统提示词；  
> 8）做静态评估；  
> 9）动态评估题必须来自 web 搜索的专业问题，不要自己编题；  
> 10）通过两轮评估后，再整理成最终交付版并归档。
