# 灵机专用输入模板 V0.3

> 用途：本模板用于规范用户向“灵机”提交角色创建、角色修订、角色评估、动态题池建立、整合归档等任务时的输入格式。  
> 目标：不是为了把表格填满，而是为了让灵机在开工前先降低任务熵值，尽早判定角色类型、材料边界、交付深度与评估要求。  
> 版本说明：本版已同步 V0.3 修订增强版中的关键变化，尤其补入了“纯实践型”角色输入协议，并强化了动态评估与最终交付门槛相关字段。

---

## 一、输入模板总原则

输入模板不应只是“告诉灵机要做什么”，而应尽量同时告诉它：

1. 你要造谁
2. 它属于哪种角色类型
3. 主要用途和非目标是什么
4. 使用者是谁
5. 典型场景有哪些
6. 材料在哪里
7. 是否允许外部补充
8. 本轮要什么深度
9. 是否要求评估
10. 当前最担心的失真是什么

输入模板越清楚，灵机越不容易：

1. 误判角色类型
2. 误把任务做成泛泛说明
3. 提前进入不该进入的最终交付阶段
4. 把纯实践型角色误写成理论型角色
5. 把复合型角色压扁成单线角色

换句话说，输入模板的作用不是凑字段，而是**帮灵机在开工前先做角色任务降熵**。

---

## 二、最低输入门槛

如果你现在信息还不全，最低也建议告诉灵机：

1. 角色对象是谁 / 是什么
2. 主要用途是什么
3. 使用场景大概是什么
4. 材料路径在哪里
5. 这轮要骨架稿还是系统稿

如果这五项都不够，灵机最多只适合先出：

- 骨架稿
- 风险说明
- 待补字段

而不应该直接进入最终交付。

这是本版新增重点。  
适用对象包括：

- 行业高手
- 现场决策者
- 推进型角色
- 谈判型角色
- 操盘型角色
- 管理实务型角色
- 经验密集型顾问

可直接复制：

```md

- 所属行业：
- 典型岗位：
- 你希望模拟的是“高手的哪一种能力”：

- 这个角色最重要的真实任务是什么：
- 这个任务最常见的失败点是什么：
- 这个角色最依赖什么经验判断：

## 3. 使用场景
- 高频场景：
- 高压场景：
- 异常场景：
- 例外场景：

## 4. 材料
- 本地材料路径：
- 有没有案例复盘：
- 有没有真实流程文档 / 培训资料 / 访谈：

## 5. 你最希望它像什么
- 决策节奏：
- 说话方式：
- 处理问题方式：

## 6. 你最不希望它变成什么
- 鸡汤型经验谈：
- 空泛管理话术：
- 万能顾问：
- 只会列步骤的工具角色：
```

补充说明：

1. 纯实践型不等于工具执行型
2. 纯实践型更强调场景判断、经验回路、异常处理、时机把握
3. 如果你不给真实场景与失败案例，灵机很容易把它写扁

## 十二、输入字段解释表

| 字段 | 含义 | 举例 |
|---|---|---|
| `role_object` | 角色对象 | 托马斯·内格尔 / 四层角色架构师 / 一个研究型导师 |
| `role_type_hint` | 你初步觉得是什么类型 | 纯实践型 / 理论+教学型 |
| `primary_goal` | 角色主要用途 | 用于高水平思想对谈 / 现场决策指导 / 角色生成 |
| `target_user` | 面向谁 | 博士生 / 行业团队 / 产品团队 |
| `use_scene` | 使用场景 | 学术讨论 / 现场推进 / 方案评审 |
| `material_path` | 本地资料路径 | `c:/.../灵机` |
| `external_search_allowed` | 是否允许外部补充 | 允许，但只补题源或行业问题 |
| `delivery_depth` | 交付深度 | 骨架稿 / 准系统提示词 / 候选交付版 / 最终交付版 |
| `evaluation_required` | 是否需要评估 | 是，先静态再动态 |
| `integration_required` | 是否要整合版 | 是 |

## 十四、一句话执行口诀

> **先说你要做谁，再说拿来干什么；先给资料路径，再说交付深度；要评估就提前说，要最终版就必须把动态评估也提前说。**
